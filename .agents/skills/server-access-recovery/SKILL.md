# Восстановление доступа к серверу (Firewall/SSH)

> [!WARNING]
> Если SSH-подключение (`ssh root@ip`) завершается ошибкой `Connection timed out`, значит порт 22 закрыт или ваш IP заблокирован файрволом (ufw/iptables).

## 1. Актуальные данные сервера

- **Основной IP**: `89.104.70.58` (Reg.ru / Cloud.reg.ru)
- **Альтернативный IP**: `176.124.214.195` (использовался ранее)
- **Пользователь**: `root`
- **Порт SSH**: `22` (стандартный)

## 2. Алгоритм восстановления через VNC-консоль

Если порт 22 заблокирован, единственный путь — **Веб-консоль (VNC)** в панели управления хостингом.

1.  **Получение ссылки**: Запросите у пользователя ссылку на VNC-консоль (например, `https://ru2-console.cloud.reg.ru/...`).
2.  **Использование Browser Agent**: Откройте ссылку через браузерного агента.
3.  **Авторизация**: Введите логин и пароль в интерактивном режиме (скриншоты помогут контролировать фокус ввода).
4.  **Разблокировка IP**:
    - Узнайте текущий исходящий IP агента: `Invoke-RestMethod -Uri https://ifconfig.me/ip`.
    - Добавьте его в белый список:
      ```bash
      ufw allow from <YOU_IP> to any port 22
      ```
    - Или откройте порт глобально (если безопасно):
      ```bash
      ufw allow 22
      ```
5.  **Проверка статуса**: `ufw status numbered`.

## 3. Правила деплоя после восстановления
- Используйте `sshpass` для автоматизации передачи файлов, если SSH-ключи не настроены.
- Сначала проверяйте соединение простой командой: `ssh -o ConnectTimeout=5 root@<IP> "echo ok"`.

## 4. Анализ наших ошибок (Lessons Learned)
- **Ошибка:** Во время текущей сессии при столкновении с ошибкой `Connection timed out` на 22 порту мы потратили более 20 минут впустую (искали конфиги Nginx, перебирали все старые IP в `.ssh/known_hosts`, сканировали порты 80, 443, 2222, искали `deploy.sh`).
- **Решение:** Если происходит `Timeout` на 22 порту сервера, который очевидно включен (работает сайт/пингуется Google), это на 99% означает, что IP-адрес агента был заблокирован или сменился, и нас не пускает Firewall (UFW/Iptables). В таких ситуациях нужно **немедленно прекращать** перебор консольных команд и **запрашивать у пользователя ссылку на VNC-консоль** хостинг-провайдера. Это экономит десятки минут.
