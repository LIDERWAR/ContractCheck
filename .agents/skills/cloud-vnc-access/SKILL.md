# Самостоятельное восстановление доступа к серверу через ЛК Cloud.reg.ru

> [!IMPORTANT]
> При получении ошибки `WinError 10060`, `Connection timed out` или любой другой блокировки по порту 22 (SSH), **НЕ НУЖНО ПРОСИТЬ ПОЛЬЗОВАТЕЛЯ СКИНУТЬ ССЫЛКУ ВО ВРЕМЯ АВТОНОМНОЙ РАБОТЫ**. 
> Изучи этот навык и используй инструмент `browser_subagent`, чтобы восстановить доступ полностью автономно!

## 1. Учетные данные от Личного Кабинета

- **URL Личного кабинета**: `https://cloud.reg.ru/`
- **Логин**: `xx-serg-xx2011@yandex.ru`
- **Пароль**: `6262662JOI`

## 2. Инструкция для Browser Subagent

При возникновении проблем с доступом к серверу по SSH, запусти `browser_subagent` с примерно такой задачей:

1. Открой браузер и перейди на страницу авторизации хостинга (проще всего найти кнопку входа на `https://cloud.reg.ru/`).
2. Введи логин `xx-serg-xx2011@yandex.ru` и пароль `6262662JOI`.
3. Дождись загрузки панели управления облачными серверами.
4. Выбери нужный сервер (основной IP: `89.104.70.58` или альтернативный `176.124.214.195`).
5. Запусти VNC-консоль сервера (web-консоль).
6. В открывшейся консоли терминала:
   - Если сервер просит Login, введи `root`.
   - Если просит Password, введи пароль от сервера (например, `xTi6i3FQgs9UCSeT`).
7. Введи команду для разблокировки твоего текущего адреса (сначала узнай его через `Invoke-RestMethod -Uri https://ifconfig.me/ip`):
   ```bash
   ufw allow from <ТЕКУЩИЙ_IP_АГЕНТА> to any port 22
   ```
8. Убедись, что правило применено (вывод вроде `Rule added`).
9. Заверши задачу браузерного агента.

После того как `browser_subagent` добавит твой IP в белый список, попытайся снова запустить деплой-скрипт или подключиться по SSH.

> [!WARNING]
> Обращайся к пользователю за помощью (согласно `project-rules`), только если во время авторизации в ЛК (Reg.ru) запросит капчу/2FA, которую ты не сможешь обойти, или если возникнет критическая ситуация, блокирующая процесс. В иных случаях действуй полностью автономно.
