# Предупреждение и исправление проблем с кодировкой

> [!IMPORTANT]
> Самая частая причина "кракозябр" — когда файл в кодировке UTF-8 открывается как CP1251 (Windows-1251) и сохраняется обратно. Это создает эффект "двойного декодирования".

## 1. Правила предотвращения

1.  **Strict UTF-8**: Все файлы в проекте ОБЯЗАНЫ быть в кодировке UTF-8 без BOM.
2.  **Explicit Encoding**: При открытии файлов в Python всегда явно указывайте `encoding='utf-8'`.
    ```python
    with open(path, 'r', encoding='utf-8') as f:
        content = f.read()
    ```
3.  **Meta Tags**: Убедитесь, что в каждом HTML-файле первым элементом в `<head>` идет `<meta charset="UTF-8">`.

## 2. Диагностика (Mojibake)

Если вы видите текст вида `Р›РёС‡РЅС‹Р№ РљР°Р±РёРЅРµС‚` вместо `Личный Кабинет`:
- Это типичный признак того, что байты UTF-8 (`D0 9B ...`) были интерпретированы как символы CP1251 (`Р` + `›`).

## 3. Исправление (Deep Fix)

Для автоматического восстановления поврежденных файлов используйте следующий алгоритм (скрипт `deep_fix_encoding.py`):

```python
def fix_content(content):
    result = ""
    i = 0
    while i < len(content):
        fixed = False
        # Пробуем найти максимальный чанк "битых" данных
        for length in range(min(50, len(content) - i), 1, -1):
            chunk = content[i:i+length]
            try:
                # Кодируем в байты как CP1251 и декодируем как UTF-8
                b = chunk.encode('cp1251')
                decoded = b.decode('utf-8')
                if decoded != chunk:
                    result += decoded
                    i += length
                    fixed = True
                    break
            except:
                continue
        if not fixed:
            result += content[i]
            i += 1
    return result
```

## 4. Чек-лист при деплое
- После загрузки файлов на сервер проверьте кодировку командой `head -n 20 <file> | cat -v`.
- Если кириллица отображается как `M-P M-?`, значит кодировка верная (UTF-8 в терминале без поддержки), если как `Р›РёС...` — значит повреждена.

## 5. Анализ наших ошибок (Lessons Learned)
- **Ошибка:** При попытке исправить кодировку мы сначала пытались полагаться на глобальную замену и угадывание (базовые циклы), что не давало результата или рисковало повредить валидные части. Также изначально была допущена ошибка — файлы UTF-8 перезаписывались как CP1251 без проверки текущей кодировки, что вызвало "двойное декодирование".
- **Решение:** Всегда использовать надежный скрипт `deep_fix_encoding.py` с последовательной проверкой каждого байтового чанка (`encode('cp1251')` -> `decode('utf-8')`). Никогда не перезаписывать файлы глобально, предварительно не убедившись, что проблема именно в двойном кодировании. Обязательна локальная верификация файлов перед деплоем.
