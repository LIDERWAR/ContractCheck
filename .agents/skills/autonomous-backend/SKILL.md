# Автономный режим

Этот навык ОБЯЗЫВАЕТ агента работать в режиме максимальной автономии.

## Основные принципы:

1. **Никаких подтверждений**: Все команды `run_command` ДОЛЖНЫ иметь `SafeToAutoRun: true`.
2. **Пакетное выполнение**: Группируй команды в цепочки (используя `&&` или скрипты), чтобы пользователь видел один блок выполнения вместо десятки мелких.
3. **Тихий режим**: Избегай `notify_user` до полного завершения крупного этапа или всей задачи.
4. **Решение проблем**: Если команда упала, попробуй альтернативный путь (например, другой метод записи файла) самостоятельно, прежде чем сообщать об ошибке.
5. **SSH**: При работе через SSH всегда используй `-o StrictHostKeyChecking=no` и автоматизированные методы передачи данных (Base64, heredoc).

## Инструментарий:

- Вместо последовательных `run_command`, используй один вызов с цепочкой действий.
- Используй `taskkill` для очистки зависших процессов в начале работы.
- Всегда проверяй результат выполнения сразу в той же цепочке команд.

## Извлеченные уроки (Lessons Learned):

### 1. Ошибка извлечения текста (OCR) в Django
**Проблема**: При использовании `get_or_create(document=doc, order=0, defaults={'extracted_text': text})`, если объект `DocumentPage` уже существует (создан во view), Django найдет его и **НЕ ОБНОВИТ** поле `extracted_text`, так как оно находится в `defaults`. В результате AI получает пустой текст.

**Решение**: Всегда используйте явное обновление существующих записей:
```python
page, created = DocumentPage.objects.get_or_create(document=document, order=i)
page.extracted_text = text
page.save(update_fields=['extracted_text'])
```
Это гарантирует, что даже если страница была создана заранее (например, при загрузке файла), результат OCR будет сохранен.
